HALLAZGOS Y CONCLUSIONES

1. Se implementó satisfactoriamente una automatización End-to-End para validar el flujo completo de compra en SauceDemo.

2. El escenario cubre las funcionalidades críticas del proceso de compra:

* Inicio de sesión
* Selección de productos
* Gestión del carrito
* Registro de información del comprador
* Confirmación de compra

3. Se aplicó el patrón Page Object Model (POM), permitiendo una mejor mantenibilidad, reutilización de código y escalabilidad de la solución.

4. Se implementó generación automática de evidencias mediante capturas de pantalla en cada etapa relevante del proceso.

5. Se implementó generación automática de reportes HTML utilizando ExtentReports, permitiendo visualizar de forma clara los resultados de ejecución y las evidencias asociadas.

6. El framework fue configurado para conservar el historial de ejecuciones, generando reportes y capturas únicas por fecha y hora.

7. Durante las pruebas realizadas no se identificaron defectos funcionales que impidieran completar el flujo de compra definido en el alcance del ejercicio.

CONCLUSIÓN GENERAL

La automatización desarrollada permite validar de manera confiable y repetible el proceso de compra de SauceDemo, reduciendo tiempos de ejecución manual, aumentando la trazabilidad de resultados y facilitando futuras ampliaciones del conjunto de pruebas automatizadas.
